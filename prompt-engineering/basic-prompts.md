# 基础提示词

掌握提示词的艺术是高效使用 AI 的关键。本课程将教您如何编写有效的提示词。

## 提示词的基本结构

### 核心要素

一个有效的提示词通常包含以下要素：

```
[角色设定] + [任务描述] + [上下文] + [约束条件] + [输出格式]
```

### 示例分析

**差的提示词：**
```
写一篇文章
```

**好的提示词：**
```
你是一位专业的科技博主。请写一篇关于"AI 如何改变创意产业"的
文章，目标读者是内容创作者。文章应该：
- 包含 3 个主要应用场景
- 每个场景有具体案例
- 字数在 800-1000 字之间
- 使用 Markdown 格式
```

## 提示词技巧

### 1. 角色设定（Persona）

给 AI 分配一个明确的角色，可以显著提升输出质量。

**示例：**

| 角色 | 示例提示词 |
|------|-----------|
| 老师 | "作为一名有 10 年经验的编程教师..." |
| 编辑 | "作为《纽约时报》的高级编辑..." |
| 顾问 | "以麦肯锡咨询顾问的视角..." |
| 创意总监 | "你是一位获得过多个奖项的创意总监..." |

### 2. 清晰的任务描述

使用具体、可衡量的语言描述任务。

**模糊 vs 具体：**

❌ "帮我优化这个网站"
✅ "帮我优化这个电商网站的产品页面，重点提高转化率，建议包括文案、布局和 CTA 位置的改进"

### 3. 提供上下文

提供足够的背景信息让 AI 理解任务。

**上下文要素：**
- 目标受众
- 业务背景
- 限制条件
- 相关数据
- 预期结果

### 4. 约束条件

明确告诉 AI 什么**不**要做。

**示例：**
- "不要使用技术术语"
- "避免过度夸张的表达"
- "不要提及竞争对手"
- "保持专业但友好的语调"

### 5. 指定输出格式

告诉 AI 您期望的输出形式。

**常见格式：**
- Markdown（文档、博客）
- JSON（数据结构）
- HTML（网页代码）
- 表格（数据对比）
- 列表（要点总结）

## 提示词模板

### 内容创作模板

```markdown
角色：[你的角色]
任务：[具体任务]
目标：[目标受众/目的]
要求：
- [要求 1]
- [要求 2]
- [要求 3]
格式：[Markdown/HTML/等]
长度：[字数/段落数]
```

### 问题解决模板

```markdown
我遇到一个问题：[问题描述]

背景信息：
- [相关背景 1]
- [相关背景 2]

请提供：
1. 问题的根本原因分析
2. 3 个可能的解决方案
3. 每个方案的优缺点
4. 推荐方案及理由
```

### 代码生成模板

```markdown
我需要[编程语言]代码来完成：[功能描述]

技术栈：
- 框架：[框架名称]
- 数据库：[数据库类型]
- 其他：[其他技术]

要求：
- 遵循最佳实践
- 包含错误处理
- 添加注释说明
- 考虑性能优化
```

## 迭代优化

### 迭代式写作

不要期望一次就能得到完美结果：

1. **初稿**：让 AI 生成初步内容
2. **反馈**：指出需要改进的地方
3. **优化**：根据反馈调整提示词
4. **精炼**：最终润色

### 示例迭代

**第一轮：**
```
写一篇关于 AI 的文章
```
**反馈：** "太宽泛了，可以具体一点"

**第二轮：**
```
写一篇关于 AI 在教育中应用的文章
```
**反馈：** "可以加入一些案例"

**第三轮：**
```
写一篇关于 AI 在教育中应用的文章，包括在线教育、个性化学习和
教师辅助三个方面的具体案例
```

## 常见错误

### 错误 1：提示词太简短

❌ "写个广告文案"
✅ "为这款智能手表写 3 个不同风格的广告文案，目标用户是 25-35 岁的城市白领"

### 错误 2：没有指定角色

❌ "解释这个代码"
✅ "作为一位资深的 Python 开发者，请解释这段代码的逻辑和优化空间"

### 错误 3：缺少示例

❌ "写个类似的标题"
✅ "写 5 个类似'人工智能如何重塑金融业'的标题风格，主题是 AI 在医疗领域的应用"

### 错误 4：一次要求太多

❌ "写一本书"（太大）
✅ "先写第一章，主题是..."（分步进行）
